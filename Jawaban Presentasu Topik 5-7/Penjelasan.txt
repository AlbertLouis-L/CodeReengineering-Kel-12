Penjelasan Topik 5 - 7 :

5 :
Divergent Changes : pada bagian class Rectangle terdapat 2 method yang tidak saling terkait maka untuk merefactornya 
bisa memisahkan methodnya ke class masing-masing.

Parallel Inheritance Hierarchy : Code smell terdapat pada 2 hierarchy yaitu Shape2D dan AreaInterface, jika membuat sebuah object baru 
maka harus membuat 2 class, untuk merefactorny kita implement kan areainterface ke dalam shape2d dan membuat methodny masing2 pada child shape2d


6 :
Comments : Penggunaan comment yang kurang optimal dan kurang jelas, untuk merefactornya kita akan langsung mengubah variable dalam code supaya namanya 
langsung dapat menjelaskan variable dan methodnya itu sendiri.

DuplicateCode : terdapat print pada semua beberapa method, maka untuk merefaktornya dapat menggunakan extract method, masing-masing method return 
sebuah nomor dan function printnya dipindahkan ke menu.


7 :
Lazy Class : cara merefaktornya adalah menambahkan sebuah class baru untuk mengutilize lazy classnya.

Data Class : 
Data class terjadi didalam class coordinates. Dimana didalamnya hanya terdapat variabel penyimpanan variabel dan setter getternya. 
untuk merefaktor codenya dapat memindahkan method calculatedistance kepada class coordinate.
